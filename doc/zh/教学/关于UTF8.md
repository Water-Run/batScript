# 编码处理（UTF-8 与 ASCII）

## 概述

`.bat` 脚本的字符编码历来是一个隐患：Windows 控制台默认代码页（通常为 GBK/CP936）与现代 UTF-8 工作流之间的冲突，常导致中文乱码或命令解析失败。

`batScript` 编译器内置了**编码自动检测与代码页切换**机制，使开发者无需手动插入 `chcp 65001` 等原生指令，即可在输出的 `.bat` 文件中获得正确的编码环境。

## 自动检测

编译器在编译阶段会对源文件（`.bs`）进行全文扫描，逐字符判断是否存在超出 ASCII 范围（码点 > 127）的字符。

**触发条件**：源文件中包含任意一个非 ASCII 字符（例如中文、日文、带音调的拉丁字母、全角符号等）。

**触发效果**：编译器在生成的 `.bat` 文件头部自动插入代码页切换指令：

```bat
@chcp 65001 >nul
```

该指令会在脚本运行时将控制台切换至 UTF-8 代码页，确保非 ASCII 内容能够正确输出与读取。

**不触发时**：若源文件全部由 ASCII 字符组成，编译器不插入任何额外指令，保持输出文件的最小化。

## `# Flag:` 文档注释超控  

当默认的自动检测行为不符合需求时，可在**文件级文档注释**（`#` 块）中使用 `# Flag: ` 指令手动指定编码策略，超控编译器的自动判断。

> 必须位于文件顶部的文档注释块内，且只应出现一次。

### `# Flag: ASCII`

强制禁用 UTF-8 模式。无论源文件中是否包含非 ASCII 字符，编译器均不插入代码页切换指令。

### `# Flag: UTF8`

强制使用 UTF-8 模式。无论源文件中是否仅包含 ASCII 字符，编译器均向插入代码页切换指令。

## `bsc -Flag` 编译器参数

除文件级`# Flag:`注释外，编译器命令行也提供了对应的 `-Flag` 参数，用于在**不修改源文件**的前提下在外部超控编码策略。

```cmd
bsc -Compile input_file -Output output_file -Flag <模式>
```

| 参数值  | 说明                                                  |
|---------|-------------------------------------------------------|
| `utf8`  | 强制启用 UTF-8 模式，等效于文件内的 `# Flag: UTF8`    |
| `ascii` | 强制禁用 UTF-8 模式，等效于文件内的 `# Flag: ASCII`   |
| `auto`  | 使用自动检测（默认行为，不指定 `-Flag` 时即为此模式） |
