# 编码规范

在编写 `batScript` 代码时，你需要遵循以下编码规范。`bsc` 会在不符合时给出警告或拒绝编译。

## 原则

编写清晰、明确、美观的代码。代码首先是给人读的，其次才是给机器运行的。在 `batScript` 中，清晰的命名、一致的格式和有意义的注释是优质代码的核心标准。

## 代码布局和括号风格

**缩进**使用 TAB，每一层代码块增加一个 TAB。不使用空格缩进，不混用空格与 TAB。

建议每行不超过 **64 个字符**。超出时应将长表达式拆分为多行，或将其提取为具名中间变量：

```batscript
; 不推荐
set result = %str%(some_long_variable + another_variable)

; 推荐
set sum = some_long_variable + another_variable
set result = %str%sum
```

> 当可自动推导时, 类型后置声明推荐省略  

`batScript` 使用 **AllMan 风格**括号：左花括号单独占一行，与对应关键字或声明同级缩进，右花括号同样单独占一行。不应将左花括号放在行尾：

```batscript
; 正确
func calculate(a: num, b: num) -> num
{
    if (a > b)
    {
        return a - b
    }
    else
    {
        return b - a
    }
}

; 错误
func calculate(a: num, b: num) -> num {
    ...
}
```

**运算符**两侧各保留一个空格，赋值运算符和比较运算符同理。函数调用的参数列表中，逗号后紧跟一个空格。括号内侧不添加空格：

```batscript
set result: num = a + b * c
set is_valid: bool = age >= 18 and age < 60
call_func(arg_one, arg_two, arg_three)

; 不推荐
if a > b 
```

**空行**方面，函数与过程声明之间留一个空行；`group` 组之间留两个空行；文件头部的文档注释与首个声明之间留一个空行：

```batscript
# Description: 示例文件
# File: Example.bs

proc first()
{
    echo "first"
}

proc second()
{
    echo "second"
}


group GroupA
{
    let VALUE_A: num = 1
}


group GroupB
{
    let VALUE_B: num = 2
}
```

行内注释 `;` 与代码之间留一个空格。注释内容应说明**为什么**，而非复述代码本身做了什么。文档注释 `#` 用于文件、组、函数和过程，所有这类结构都应当编写文档注释，缺少时 `bsc` 会给出警告：

```batscript
; 不推荐：重复代码内容
set count = count + 1 ; count 加一

; 推荐：说明意图
set count = count + 1 ; 跳过已处理的首条记录
```

## 标识符命名规范

不同类型的标识符使用不同的命名风格，这是 `batScript` 编码规范的核心，混用风格会导致 `bsc` 发出警告。

| 类别        | 风格                        | 示例                         |
|-------------|-----------------------------|------------------------------|
| 文件名      | 大驼峰（PascalCase）        | `HelloWorld.bs`              |
| 变量 / 参数 | 小写 + 下划线（snake_case） | `user_name`，`raw_input`     |
| 常量        | 全大写 + 下划线             | `MAX_COUNT`，`DEFAULT_NAME`  |
| 函数 / 过程 | 小驼峰（camelCase）         | `getUserName`，`printReport` |
| 组名        | 大驼峰（PascalCase）        | `MathUtils`，`AppConfig`     |
| 配置名      | 每词首字母大写 + 下划线     | `Max_Retry_Count`            |

变量命名**区分大小写**。`user_name` 和 `User_name` 是两个不同的标识符，但这类命名不符合规范，`bsc` 会给出警告。参数命名与普通变量相同，使用 snake_case：

```batscript
let MAX_RETRY: num = 3
set user_input: str = ""

func parseInput(raw_text: str, max_len: num) -> bool
{
    ...
}

group InputHandler
{
    ...
}
```

## 名称语义建议

**布尔类型**变量和参数推荐使用 `is_`、`has_`、`can_` 等前缀，使其语义自明：

```batscript
; 推荐
set is_admin: bool = False
set has_error: bool = True
set can_proceed: bool = ?input_val

; 不推荐
set admin: bool = False
set flag: bool = True
```

**数组**变量应使用复数形式命名，体现其集合性质。`pair<K, V>` 的变量名应反映其包含的两端含义，而不应使用无意义的单字母或加类型后缀：

```batscript
; 推荐
set scores: arr<num> = [90, 85, 78]
set user_names: arr<str> = ["Alice", "Bob"]
set name_age: pair<str, num> = ("Alice", 20)

; 不推荐
set score_arr: arr<num> = [90, 85, 78]  ; 不必要的类型后缀
set p: pair<str, num> = ("Alice", 20)   ; 无意义的命名
```

函数与过程的命名应清晰表达行为，通常以动词开头：

```batscript
func calculateAverage(scores: arr<num>) -> num { ... }
func isValidInput(raw: str) -> bool { ... }
proc printReport(title: str) { ... }
proc saveToFile(path: str, content: str) { ... }
```

除极为通用的缩写（如 `id`）外，不应使用缩写命名。名称的长度应由其语义决定，而非追求简短：

```batscript
; 不推荐
set usr_nm: str = "Alice"
set cnt: num = 0
set tmp: str = ""

; 推荐
set user_name: str = "Alice"
set retry_count: num = 0
set placeholder: str = ""
```

## 作用域和可见性相关命名

`batScript` 不使用 `g_` 等前缀区分全局和局部变量，作用域完全依赖上下文判断。因此，命名应足够具体，使读者无需追溯定义便能理解变量所属的上下文。

在局部作用域（函数或过程内）访问全局变量时，使用 `$var` 语法显式引用，并建议对其赋予更局部化的具名变量，避免直接在局部代码中散落全局引用：

```batscript
set app_version: str = "1.0.0"

proc logVersion()
{
    set version: str = $app_version  ; 显式引入全局变量
    echo `version: {version}`
}
```

`group` 成员的命名规范与普通变量和常量相同——变量用 snake_case，常量用 UPPER_SNAKE_CASE。`group` 名称本身已提供命名空间，成员名称**不需要**重复组名：

```batscript
; 不推荐：成员名重复了组名
group AppConfig
{
    let APP_CONFIG_VERSION: str = "1.0.0"
    set app_config_mode: str = "release"
}

; 推荐
group AppConfig
{
    let VERSION: str = "1.0.0"
    set mode: str = "release"
}

; 访问时语义自然清晰
echo AppConfig.VERSION
echo AppConfig.mode
```

## 反例和不推荐写法

**单字母或无意义命名**是最常见的问题，会使代码失去自描述能力：

```batscript
; 不推荐
set a: num = 10
set x: str = getin "Input>>"

; 推荐
set max_attempts: num = 10
set raw_input: str = getin "Input>>"
```

**错误的命名风格**会导致 `bsc` 警告，以下为几类典型错误：

```batscript
; 变量使用了大驼峰
set UserName: str = "Alice"

; 常量使用了小写
let max_count: num = 100

; 函数使用了下划线风格
func get_user_name(id: num) -> str { ... }

; 利用大小写敏感创建歧义对（合法但极易混淆）
set value: num = 1
set Value: num = 2
```

**忽略可空性标注**会导致编译错误，而不必要的可空标注则会削弱类型系统的保护：

```batscript
; 错误：不可空类型赋值 Nil
set age: num = Nil

; 正确
set age: num? = Nil

; 不推荐：实际永不为 Nil 却标注可空
set name: str? = "Alice"
```

**用 `set` 声明真正不变的值**会失去编译器的不变性保护，凡是整个程序生命周期内不应改变的值都应使用 `let`：

```batscript
; 不推荐
set MAX_RETRY: num = 3

; 推荐
let MAX_RETRY: num = 3
```

**过度依赖 `runraw`** 会绕过类型系统和编译检查，产生难以维护的代码。`runraw` 仅用于处理 `batScript` 确实无法覆盖的极端底层场景：

```batscript
; 不推荐：可以用 batScript 实现却使用 runraw
runraw
{
    "set /a result = 1 + 2",
    "echo %result%"
}

; 推荐
set result: num = 1 + 2
set result_txt: str = %str%result
echo result_txt
```

**不编写文档注释**会使代码难以维护，`bsc` 对所有未附文档注释的文件、`group`、`func` 和 `proc` 均会给出警告：

```batscript
; 不推荐
proc sendEmail(to: str, body: str)
{
    ...
}

; 推荐
# Description: 向指定地址发送纯文本邮件
# Param [to]: 收件人地址
# Param [body]: 邮件正文
proc sendEmail(to: str, body: str)
{
    ...
}
```
